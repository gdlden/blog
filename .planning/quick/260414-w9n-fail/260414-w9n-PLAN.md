# Quick Task 260414-w9n: 都正常保存了怎么还是提示新建失败

## Task

Fix frontend Axios interceptor rejecting successful backend responses that lack a `code` field.

## Problem

Backend protobuf endpoints (post, debt, etc.) return data directly without a `{code: "200"}` wrapper. The interceptor in `src/utils/request.ts` rejects any response where `res.data.code !== "200"`, causing the UI to show "创建失败" even though the backend successfully saved the data.

## Fix

Modify `price_recorder_vue/src/utils/request.ts`:
- Treat missing `code` as success (not just `code === "200"`)

- Run `pnpm test:unit` in `price_recorder_vue/`
