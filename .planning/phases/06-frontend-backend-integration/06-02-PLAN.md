  <action>
    Create the blog Pinia store at price_recorder_vue/src/stores/blogStore.ts following the userStore.ts pattern.

    Use Setup Store pattern (Composition API style):

    State (refs):
    - posts: ref<Post[]>([])
    - currentPost: ref<Post | null>(null)
    - loading: ref(false)
    - error: ref<string | null>(null)
    - total: ref(0)
    - currentPage: ref(1)
    - pageSize: ref(10)

    Getters (computed):
    - postCount: computed(() => posts.value.length)
    - totalPages: computed(() => Math.ceil(total.value / pageSize.value))

    Actions (async functions):
    1. fetchPosts(page?: number, size?: number): Promise<void>
       - Set loading=true, error=null
       - Call blogApi.getPosts(String(page), String(size))
       - Update posts, total, currentPage, pageSize
       - Set loading=false
       - On error: set error message

    2. createPost(data: { title: string; content: string }): Promise<void>
       - Set loading=true
       - Call blogApi.createPost(data)
       - Call fetchPosts() to refresh list
       - Set loading=false
       - On error: throw error for component to handle

    3. updatePost(id: string, data: { title: string; content: string }): Promise<void>
       - Set loading=true
       - Call blogApi.updatePost(id, data)
       - Call fetchPosts() to refresh list
       - Set loading=false
       - On error: throw error for component to handle

    4. deletePost(id: string): Promise<void>
       - Set loading=true
       - Call blogApi.deletePost(id)
       - Filter posts array to remove deleted post
       - Set loading=false
       - On error: throw error for component to handle

    5. setCurrentPost(post: Post | null): void
       - Set currentPost.value = post

    Return object with all state, getters, and actions.

    Per D-08 through D-10: Pinia store manages list data, current detail, loading states, errors.
  </action>
